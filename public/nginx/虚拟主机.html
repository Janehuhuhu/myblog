    # nginx虚拟主机

    # 简介

    虚拟主机 就是把一台物理服务器划分成多个“虚拟”的服务器，每一个虚拟主机都可以有独立的域名和独立的目录，可以独立发布一个网站。

    # 基于IP的虚拟主机

    1. 应用场景
      IP充足的环境，每个网站需要一个IP地址

    2. 特点

    • 不同IP对应不同网站
    • 访问方便，用户直接使用默认端口即可访问
    • 服务器需要有多个IP地址（一个公网IP大概一年的费用是600左右）
    • 维护方便，基于独立IP的站点，便于监控、维护。
    1. 接入方式
      终端输入ifconfig 查看所有可用的IP
    server {
    listen       127.0.0.1:80;
    server_name  localhost;
    location / {
        root   /Users/jane/nginx/test01;
        index  index.html index.htm;
    }
}
server {
    listen       192.168.1.11:80;
    server_name  localhost;
    location / {
        root   /Users/jane/nginx/test02;
        index  index.html index.htm;
    }
}

    # 基于端口的虚拟主机

    1. 应用场景
      IP不足的环境，多个网站发布使用该配置方法只需要一个IP，节省IP地址。端口无法告诉公网用户，所以不适用于公网客户，适合内部用户

    2. 特点

    • 不同端口对应不同网站
    • 访问需要加端口
    • 节省IP地址
    • 适合私网运行
    1. 接入方式
    server {
    listen       8081;
    server_name  localhost;
    location / {
        root   /Users/jane/nginx/test01;
        index  index.html index.htm;
    }
}
server {
    listen       8082;
    server_name  localhost;
    location / {
        root   /Users/jane/nginx/test01;
        index  index.html index.htm;
    }
}

    # 基于域名的虚拟主机

    1. 应用场景
      一个网站需要有一个域名，目前公网发布网站的首选

    2. 特点

    • 不同域名对应不同网站
    • 需要多个域名 可以是二级或三级域名
    • 每个站点使用默认端口，方便用户访问
    • 只需要一个IP地址，节约成本
    • 适合公网环境
    1. 接入方式
      修改/etc/hosts文件使域名和IP映射，如下：
    192.168.1.11    www.test01.com
127.0.0.1       www.test02.com

    server {
  listen       80;
  server_name  web1.zutuanxue.com;

  location / {
    root   html/web1;
    index  index.html index.htm index.php;

  }
}
server {
  listen       80;
  server_name  web2.zutuanxue.com;

  location / {
    root   html/web2;
    index  index.html index.htm;
  }
}
